### Project CodeLeap

This project is a challenge for a Frontend position at CodeLeap, the idea here was to create an application in the form of a social network, with a simple login, post visualization, and the possibility to create your posts.

The challenge behind this was to create a complete CRUD using a ready-made API that was made available by CodeLeap, creating posts and sending them to the database, reading the data of these posts and rendering them on the screen, having the possibility to edit the title and content of the posts, and last but not least, the ability to delete posts.

According to the proposed rule, login only needs a username, you can view all posts, but you can edit or delete only your posts.

---

</br>

### Functionalities

- On the first screen we will have our login, a simple registration using a username, the access button will only be available if any name is typed in the correct field.
- Once logged in, we will be able to view the posts and create our own.
- To create a post, you need to write the title and content in the correct fields, just like in our login, the button to create posts will only be available after all information is filled in.
- The name used in the login will automatically be associated with your post.
- On the home page we can also log off if necessary.
- You can see all the posts, but your own will have two icons next to the title, a button to delete, and another one to edit your post.
- The delete button will bring up a message on the screen, asking if you are sure of your choice, in case of confirmation your post will be deleted and a success message will appear on the screen.
- The edit button will open a modal with the fields for editing, the same as the common post, title, and content. This time the save changes button will be available, but you will still have to fill in all the fields to be able to edit.
- Exiting the modal by clicking outside it or canceling the post edition is possible.
- At the end of the page we have our pagination, being possible to access old posts by choosing the page number.
- Each page will display 10 posts.

---

</br>

### Possible future additions

- Like and dislike button, limited to 1 or -1 per user, per post.
- Comments inside the posts.
- Like and dislike buttons in the comments, working the same as in the posts.
- Adding images to posts.
